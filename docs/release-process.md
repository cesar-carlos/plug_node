# Release Process

## Official flow

The official publish path is GitHub Actions.

1. Implement the change.
2. Run `npm run verify`.
3. Run `npm run pack:check`.
4. Add a changeset when package behavior changes.
5. Merge to `main`.
6. The `Publish` workflow creates or updates the version PR.
7. Merge the version PR to publish packages to npm.

## Packages

- `n8n-nodes-plug-database`
  - public REST-only package
- `n8n-nodes-plug-database-advanced`
  - advanced REST + Socket package

## Notes

- the public package is the only n8n verification candidate
- trusted publishing on npm should point to `publish.yml`
- provenance is generated by the GitHub Actions publish workflow
